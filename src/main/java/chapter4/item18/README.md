# 상속보다는 컴포지션을 사용하라

### 컴포지션 이란?
* 기존 클래스가 새로운 클래스의 구성요소로 쓰이는 것

### 상속은 캡슐화를 깨뜨린다.
* 상위 클래스가 어떻게 구현되느냐에 따라 하위 클래스의 동작에 이상이 생길 수 있다.
* 대신 컴포지션과 전달 사용하자.
  * 전달 : 기존 클래스의 대응하는 메서드를 호출해 그 결과를 반환하는 것.  
  * 전달 메서드 : 새 클래스의 메서드들

### 상속 생성 규약
* 구체 클래스 각각을 따로 확장해야 한다.
* 지원하고 싶은 상위 클래스의 생성자 각각에 대응하는 생성자를 별도로 정의해줘야 한다.
* is-a 관계일 때만 사용한다.

### 컴포지션 장점
* 어떠한 구현체라도 계측할 수 있다.
* 기존 생성자들과도 함께 사용할 수 있다.
